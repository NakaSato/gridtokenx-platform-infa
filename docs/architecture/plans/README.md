## 📝 Plan Lifecycle

1. **Proposed** → Initial proposal with design and rationale
2. **In Review** → Under technical review by engineering team
3. **Approved** → Accepted for implementation
4. **In Progress** → Being implemented
5. **Completed** → Merged and deployed
6. **Deferred** → Postponed for future consideration

## 🎯 Creating a New Plan

### Template Structure
```markdown
# Feature/Plan Title

**Date:** YYYY-MM-DD
**Author:** Your Name
**Status:** Proposed

---

## Summary
Brief description of the feature/plan.

## Motivation
Why this feature is needed.

## Design
Technical design and implementation details.

## Alternatives Considered
Other approaches and why they were rejected.

## Migration Plan
Steps for implementation (if applicable).

## References
Links to related documents, issues, PRs.
```

### Process
1. Create markdown file in this directory: `YYYY-MM-DD-feature-name.md`
2. Follow the template structure above
3. Reference the plan in relevant documentation
4. Submit for review via pull request
5. Update status as plan progresses
